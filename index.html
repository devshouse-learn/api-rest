¿Qué es una API REST?

API REST (Representational State Transfer) es un estilo de arquitectura para diseñar servicios web que permite la comunicación entre diferentes sistemas a través del protocolo HTTP.

Conceptos Clave:

  • Recursos: Todo en REST es un recurso (usuarios, productos, pedidos, etc.)
  • HTTP: Utiliza los métodos estándar HTTP (GET, POST, PUT, DELETE)
  • Sin Estado: Cada petición es independiente y contiene toda la información necesaria
  • JSON/XML: Los datos se intercambian generalmente en formato JSON

Métodos HTTP Principales

GET

Propósito: Obtener/Leer datos

Ejemplo:

GET /api/users/123

Recupera información sin modificar nada en el servidor

POST

Propósito: Crear nuevos recursos

Ejemplo:

POST /api/users

Envía datos para crear un nuevo registro

PUT

Propósito: Actualizar recurso completo

Ejemplo:

PUT /api/users/123

Reemplaza completamente un recurso existente

DELETE

Propósito: Eliminar recursos

Ejemplo:

DELETE /api/users/123

Elimina un recurso del servidor

Ejemplo Práctico

// Ejemplo de petición GET con fetch
fetch('https://api.ejemplo.com/users')
  .then(response => response.json())
  .then(data => {
    console.log('Usuarios:', data);
  })
  .catch(error => {
    console.error('Error:', error);
  });

// Ejemplo de petición POST
fetch('https://api.ejemplo.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    nombre: 'Juan',
    email: 'juan@ejemplo.com'
  })
})
  .then(response => response.json())
  .then(data => console.log('Usuario creado:', data));
# Ejemplo de petición GET con requests
import requests

response = requests.get('https://api.ejemplo.com/users')
usuarios = response.json()
print('Usuarios:', usuarios)

# Ejemplo de petición POST
nuevo_usuario = {
    'nombre': 'Juan',
    'email': 'juan@ejemplo.com'
}
response = requests.post(
    'https://api.ejemplo.com/users',
    json=nuevo_usuario
)
print('Usuario creado:', response.json())

Códigos de Estado HTTP

2xx - Éxito

  • 200 OK: Petición exitosa
  • 201 Created: Recurso creado
  • 204 No Content: Éxito sin contenido

4xx - Error del Cliente

  • 400 Bad Request: Petición inválida
  • 401 Unauthorized: No autenticado
  • 404 Not Found: Recurso no encontrado

5xx - Error del Servidor

  • 500 Internal Server Error: Error del servidor
  • 503 Service Unavailable: Servicio no disponible

Autenticación en APIs REST

API Keys

Una clave única que identifica al cliente

Authorization: ApiKey abc123xyz
  • ✓ Fácil de implementar
  • ✗ Menos seguro que otros métodos

Bearer Token (JWT)

Token firmado que contiene información del usuario

Authorization: Bearer eyJhbGci...
  • ✓ Muy seguro
  • ✓ Stateless (sin estado)
  • ✓ El más usado actualmente

OAuth 2.0

Protocolo de autorización para aplicaciones de terceros

  • ✓ Muy seguro
  • ✓ Usado por Google, Facebook, etc.
  • ✗ Más complejo de implementar

Entendiendo JWT

Headers HTTP Importantes

Content-Type

Content-Type: application/json

Indica el tipo de contenido que se envía

Accept

Accept: application/json

Indica qué tipo de contenido acepta el cliente

Authorization

Authorization: Bearer token123

Contiene las credenciales de autenticación

Cache-Control

Cache-Control: no-cache

Controla el comportamiento del caché

Ejemplo con Autenticación

// Login y obtención de token
async function login(username, password) {
  const response = await fetch('https://api.ejemplo.com/login', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ username, password })
  });
  
  const data = await response.json();
  return data.token;
}

// Usar el token en peticiones protegidas
async function obtenerDatosProtegidos(token) {
  const response = await fetch('https://api.ejemplo.com/datos', {
    headers: {
      'Authorization': `Bearer ${token}`,
      'Accept': 'application/json'
    }
  });
  
  return await response.json();
}

// Uso
const token = await login('usuario', 'password');
const datos = await obtenerDatosProtegidos(token);
# Login y obtención de token
import requests

def login(username, password):
    response = requests.post(
        'https://api.ejemplo.com/login',
        json={'username': username, 'password': password}
    )
    return response.json()['token']

# Usar el token en peticiones protegidas
def obtener_datos_protegidos(token):
    headers = {
        'Authorization': f'Bearer {token}',
        'Accept': 'application/json'
    }
    response = requests.get(
        'https://api.ejemplo.com/datos',
        headers=headers
    )
    return response.json()

# Uso
token = login('usuario', 'password')
datos = obtener_datos_protegidos(token)

CORS (Cross-Origin Resource Sharing)

CORS es un mecanismo de seguridad que permite o restringe peticiones desde diferentes dominios.

Problema:

ejemplo.com

api.otrodominio.com

🚫

Por defecto, los navegadores bloquean peticiones entre diferentes dominios

Solución - Headers CORS:

Access-Control-Allow-Origin: * Access-Control-Allow-Methods: GET, POST, PUT, DELETE Access-Control-Allow-Headers: Content-Type, Authorization

CORS Explicado

Rate Limiting

Limita el número de peticiones que un cliente puede hacer en un período de tiempo.

Headers de Rate Limit:

X-RateLimit-Limit: 100 X-RateLimit-Remaining: 95 X-RateLimit-Reset: 1634567890

Respuesta cuando se excede:

Status: 429 Too Many Requests Retry-After: 3600

Beneficios:

  • 🛡️ Protege contra ataques DDoS
  • ⚡ Previene sobrecarga del servidor
  • 📊 Control de uso del API
  • 💰 Base para modelos de pricing

Mejores Prácticas

Usa nombres en plural para recursos

✓ /api/users ✗ /api/user

Usa sustantivos, no verbos

✓ GET /api/users ✗ GET /api/getUsers

Usa códigos de estado HTTP correctos

200 para éxito, 404 para no encontrado, 500 para errores del servidor

Implementa paginación para listas grandes

GET /api/users?page=1&limit=20

Documenta tu API

Usa herramientas como Swagger/OpenAPI

Implementa HATEOAS

Incluye links a recursos relacionados en las respuestas

Ejemplo Completo de API

const express = require('express');
const cors = require('cors');
const rateLimit = require('express-rate-limit');

const app = express();

// Middleware
app.use(cors());
app.use(express.json());

// Rate Limiting
const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutos
  max: 100 // límite de 100 peticiones
});
app.use('/api/', limiter);

// Rutas
app.get('/api/v1/users', (req, res) => {
  const page = parseInt(req.query.page) || 1;
  const limit = parseInt(req.query.limit) || 10;
  
  // Simulación de datos
  const users = [
    { id: 1, name: 'Juan', email: 'juan@ejemplo.com' },
    { id: 2, name: 'María', email: 'maria@ejemplo.com' }
  ];
  
  res.json({
    data: users,
    pagination: {
      page,
      limit,
      total: users.length
    },
    _links: {
      self: `/api/v1/users?page=${page}`,
      next: `/api/v1/users?page=${page + 1}`
    }
  });
});

app.post('/api/v1/users', (req, res) => {
  const { name, email } = req.body;
  
  if (!name || !email) {
    return res.status(400).json({
      error: 'Nombre y email son requeridos'
    });
  }
  
  const newUser = { id: 3, name, email };
  res.status(201).json(newUser);
});

app.listen(3000, () => {
  console.log('API corriendo en puerto 3000');
});

CORS y Headers HTTP

Control de acceso entre orígenes y cabeceras importantes

¿Qué es CORS?

CORS (Cross-Origin Resource Sharing) es un mecanismo de seguridad que permite a un servidor indicar qué orígenes (dominios) tienen permiso para acceder a sus recursos.

Headers CORS Principales

  • Access-Control-Allow-Origin: Especifica qué dominios pueden acceder
  • Access-Control-Allow-Methods: Métodos HTTP permitidos
  • Access-Control-Allow-Headers: Headers personalizados permitidos
  • Access-Control-Allow-Credentials: Permite envío de cookies

Headers HTTP Importantes

Content-Type

application/json

Indica el tipo de contenido enviado

Authorization

Bearer token123

Contiene credenciales de autenticación

Accept

application/json

Tipo de respuesta esperada

User-Agent

Mozilla/5.0...

Identifica el cliente que hace la petición

Ejemplo de CORS en Node.js

// Configuración CORS en Express
const express = require('express');
const cors = require('cors');
const app = express();

// Permitir todos los orígenes
app.use(cors());

// O configuración personalizada
app.use(cors({
  origin: 'https://miapp.com',
  methods: ['GET', 'POST', 'PUT', 'DELETE'],
  allowedHeaders: ['Content-Type', 'Authorization'],
  credentials: true
}));

// Ruta de ejemplo
app.get('/api/data', (req, res) => {
  res.json({ message: 'Datos con CORS habilitado' });
});
